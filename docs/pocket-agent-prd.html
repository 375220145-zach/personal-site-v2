1. Summary


Pocket Agent 是一个嵌入手机的零配置 LLM agent。用户说话即可让 AI 在手机上执行跨应用操作——发文件、搜机票、设闹钟、填报销。不绑定特定 OS，数据默认留在设备端。面向全球市场，首发 Android。

3. Background


Context：桌面端有 Claude Code、Copilot 等 agent 工具，手机端仍以对话 AI 为主。能执行操作的系统级 agent 正在兴起，但现有方案要么封闭（豆包/厂商内置），要么需部署配置（AutoGLM）。


Why now：NPU 算力达标（高通 8 Gen 4 NPU 45 TOPS）；Android 17 原生 Computer Control API；豆包 7,871 万 DAU 验证需求；国务院"人工智能+"2027 智能终端普及率 >70%。


Newly possible：端侧 3B 模型可实时运行。


不做会怎样：错过手机 Agent 从"聊天工具"到"任务执行者"的范式转移窗口。豆包/厂商 Agent 会吃掉大部分用户，第三方独立 Agent 的机会窗口在 2026-2027。



4. Objective


做一个零配置、跨平台、隐私优先的手机 Agent。


Key Results (SMART)：

- KR1: MVP 3 个月内上线 Android，5 个核心任务端到端成功率 >80%

- KR2: 首次用户体验（Time to First Success）<3 分钟

- KR3: 端侧推理占比 >70%，云端仅复杂任务 fallback

- KR4: 30 日留存 >30%（对标豆包 44.5%）

验收标准（F1 示例，全功能 GWT 见详细规格）：


场景：正常流程（中文语音）
GIVEN 用户在首页，网络正常 WHEN 说"帮我把上次PDF发给张三"
THEN 3 秒内显示解析结果：意图=发文件，文件=最近PDF，收件人=张三

场景：错误处理（语音不清）
GIVEN 用户说话含糊 WHEN 识别失败
THEN 显示"没听清楚" + 文字输入框作为 fallback

场景：边界情况
GIVEN 用户说不支持的操作 WHEN Agent 无法执行
THEN 显示原因 + 3 个相近合法建议

场景：空状态
GIVEN 首次打开 WHEN 看到首页
THEN 麦克风脉冲动画 + 3 个示例任务 chips

每功能覆盖 5 种状态：空/加载/成功/错误/边界。
